Get started

Three steps. No jargon.

1

Install Node.js

Loop CMS runs on Node.js. If you don't have it, download it from nodejs.org (pick the big green button).

2

Download

Download loopcms.js and put it somewhere you'll remember — your Desktop works fine.

3

Run it

Open Terminal (Mac: Cmd+Space, type Terminal) or Command Prompt (Windows: search for cmd). Type these two lines:

$cd ~/Desktop $node loopcms.js

Open http://localhost:3000 in your browser. Log in with admin / admin. You're in.

Why this exists

Bluehost lost everything.

A year of essays. No backups. No apology. Just a support ticket that closed itself before anyone could open it again. The kind of loss that turns into a quiet promise you make to yourself in the dark.

Loop CMS is what the replacement should have been — a single file of JavaScript that runs on one command. A SQLite database you can hold in your hand. An append-only ledger underneath where every change is provable. A system that assumes the host will fail, and plans around it.

It's the thing a stranger can clone in sixty seconds and trust on day one.

What's inside

Eight ideas, one file.

Proof Vault

Every auth event, content change, and system transition appends to a hash-chained ledger. Each entry carries the previous one's hash. You can prove what happened.

Time Travel

Any content URL accepts ?at=<timestamp>. Read the site as it existed at that instant. The response tells you how far back the archive goes.

Seismograph

Before you publish, the system shows you every webhook that will fire, every cache that will invalidate, every feed that will regenerate. No surprises downstream.

Déjà Vu

Write a draft that overlaps an existing article and the system notices. A dismissible sidebar note — advisory, not a block. The draft always saves first.

Constellation Fingerprint

Every running instance gets a human name — amber-lighthouse-42, winter-temple-88. You know which version is on which server without reading a git sha.

Stranger Walk

On demand, the system walks its own sitemap, follows its own links, and checks its own images. Too many broken, and it flags itself DEGRADED.

Content Sanitization

Script tags, iframes, event handlers, javascript: URIs — stripped on write. JPEGs lose their EXIF. SVGs lose their scripts. The editor tells you what was cleaned.

REST API

/api/content, /api/search, /api/media. JWT-authenticated, CSRF-gated, rate-limited, paginated. RSS, Atom, and sitemap.xml included.

A short list

This CMS will never:

  • be boring.
  • silently lose data.
  • gate security behind a paywall.
  • make you fight the tool.
