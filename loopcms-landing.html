Why this exists

Before the word "blog" existed, there was a hand-rolled website. It lived on shared hosting for a year or two. Then the hosting company lost all the data. Not hacked. Not corrupted. Lost. Every page, every post, every word.

That was the first wound. Every CMS since has been a compromise between power and friction. Complicated setup. Complicated staging. Complicated deployment. The tools that protect your data make you fight to use them. The tools that are easy to use don't protect your data.

Loop CMS exists because data loss is betrayal, complexity is friction, and beauty is non-negotiable.

What you get

Five buttons. Everything underneath.

Proof Vault

Append-only audit trail with a hash chain. Every write is logged, sequenced, and tamper-evident. The system that lost everything once made a promise: never again.

Time Travel

See your entire site as it looked last Tuesday. Not a backup restore — a read query. Add ?at=2026-03-15 to any API call.

Content Seismograph

Before you press Publish, see what's about to happen. Sitemap update, RSS feed, webhook targets, cache invalidation — all visible before you confirm.

Déjà Vu

At 500 articles across 3 editors, nobody remembers everything. The CMS catches near-duplicates before you publish them.

Constellation Fingerprint

Every deployment gets a name you can remember. Not "version 47." amber-lighthouse-42. Say it in a meeting. Everyone knows what you mean.

Stranger Walk

The CMS navigates its own public surface on a schedule. Every URL, every image. Broken references caught before any visitor sees them.

Content Sanitization

Paste anything. Script tags, iframes, event handlers — stripped and cleaned. The CMS tells you what it did without scolding. Every input, every time.

Headless REST API

The same content that powers the admin UI is available as JSON. curl /api/content/articles. Pagination, field selection, full-text search, temporal queries.

What we refuse

This CMS will never:

Be boring.

Content Weather reads like prose. Deployments have names. The Publish button shows you what's about to happen. Delight is structural, not decorative.

Silently lose data.

The Proof Vault is append-only with a hash chain. Revisions cap but never discard without notice. The architecture that lost sheagunther.org would be structurally impossible here.

Gate security behind a paywall.

Content sanitization, CSRF protection, rate limiting, RBAC, EXIF stripping, SVG sanitization — all ship in the single file. Security is a floor, not a feature tier.

Make you fight the tool.

One file. One command. Five buttons. No plugin conflicts. No database migration. No PHP version mismatch. No lost weekends.
